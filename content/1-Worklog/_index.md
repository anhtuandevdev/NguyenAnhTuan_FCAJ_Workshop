**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [Requirement analysis and system architecture design](1.2-week2/)

**Week 3:** [Setting up identity and access management with AWS IAM](1.3-week3/)

**Week 4:** [Designing and implementing network infrastructure via Amazon VPC](1.4-week4/)

**Week 5:** [Managing costs and configuring alerts with AWS Budgets](1.5-week5/)

**Week 6:** [Deploying compute instances and telemetry simulation on EC2](1.6-week6/)

**Week 7:** [Configuring object storage and data lifecycle on Amazon S3](1.7-week7/)

**Week 8:** [Building automated backup and recovery plans with AWS Backup](1.8-week8/)

**Week 9:** [Resource categorization and group management via AWS Resource Groups](1.9-week9/)

**Week 10:** [Automating infrastructure scheduling using AWS Lambda and EventBridge](1.10-week10/)

**Week 11:** [Implementing elasticity and load balancing with Auto Scaling and ELB](1.11-week11/)

**Week 12:** [Centralized security auditing with AWS Security Hub and project teardown](1.12-week12/)
